# Quick Task 260618-d3j: Fix perpetual-loading deadlock (core signing loop)

## Symptom

Uploading any PDF (or image) hangs on a perpetual loading spinner. The core
value flow — upload → place signature → download — never starts. Reproduces
locally and on the deployed site, for every file.

## Root cause (confirmed)

A view-state deadlock between the store and the App view-router:

- `documentStore.loadDocument(url)` set `view: 'loading'`.
- `App.tsx` mounts `<DocumentViewer>` ONLY when `view === 'loaded'`; for
  `'loading'` it renders a bare `<LoadingSpinner>`.
- The ONLY transition into `'loaded'` is `setNumPages`, called exclusively from
  `<Document onLoadSuccess>` **inside `DocumentViewer`**.

So `'loading'` gates `DocumentViewer` out of the tree → react-pdf's `<Document>`
never mounts → `onLoadSuccess` (and `onLoadError`) never fire → `view` is stuck
at `'loading'` forever. `DocumentViewer` already renders its own
`<Document loading={<LoadingSpinner/>}>` for parse-time, so the top-level
`'loading'` view was both redundant and fatal for PDFs.

**Why it shipped:** the end-of-Phase-2 browser verification of the signing loop
was deferred and never performed; every `DocumentViewer` unit test force-set
`view: 'loaded'` directly, and `documentStore.test.ts` asserted the buggy
`'loading'` transition — so the deadlock was invisible to the suite.

## Tasks

1. **Fix the transition** — `src/store/documentStore.ts`
   - `loadDocument`: `view: 'loading'` → `view: 'loaded'` (+ defensive comment).
   - The top-level `'loading'` view remains, used only for the async image-wrap
     gap where `UploadZone` calls `setView('loading')` before a `docUrl` exists.

2. **Correct the bug-encoding unit test** — `src/test/documentStore.test.ts`
   - "transitions to loading" → asserts `view === 'loaded'`; renamed + commented.

3. **Add an integration regression test** — `src/test/uploadFlow.test.tsx`
   - Render the REAL `<App>` router, drive `loadDocument`, assert the react-pdf
     `<Document>` mounts (not a bare spinner) and that `onLoadSuccess` completes
     the loop (`'loaded'` + page wrappers render). This is the layer prior tests
     bypassed.

4. **Verify** — full suite green; prove the regression test FAILS on the old
   one-line code (teeth); real-browser smoke test of an actual PDF upload.

## Constraints

- Minimal change — do not restructure the view state machine.
- Must also fix the corrupt-PDF hang (onLoadError likewise never fired before).

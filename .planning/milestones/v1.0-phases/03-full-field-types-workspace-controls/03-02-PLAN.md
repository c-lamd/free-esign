<objective>
Make all five field types placeable and editable in the browser: build the FieldPalette, migrate the placement flow from `placementMode` to `armedFieldType`, render each type in the PlacedFieldWidget (image / inline input / checkbox X), and host inline text/date editing — completing the observable "arm a type → click to drop → edit → it's ready to download" loop for text, date, and checkbox (and keeping signature working).

This is the vertical slice that turns Plan 01's data/export spine into a usable workspace. After this plan a user can place a date defaulting to today, type into a text field, drop a checkbox, and (with Plan 01's export) download a PDF containing them. It also wires undo history at the call sites (drag/resize/blur push one entry each), so undo/redo is fully functional once Plan 04 adds the controls.

Per CONTEXT.md Area 1 (field palette, date default, inline text, checkbox X), the UI-SPEC FieldPalette/PlacedFieldWidget/PlacementModeOverlay contracts, and RESEARCH Sections 5-7.

NOTE: This plan assumes effectiveScale === dims.scale (zoom is added in Plan 03). Use `dims.scale` for placement geometry in THIS plan exactly as Phase 2 did; Plan 03 swaps it for `effectiveScale`. Do not introduce zoom here.

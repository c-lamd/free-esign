<objective>
Close the loop end-to-end: a "Download PDF" button in the TopBar exports the current document's original bytes with the placed signatures overlaid, and downloads `{original-name}-signed.pdf` — staying on the document afterward (LOCKED CONTEXT.md: Export / Download). This wires together Plan 01's export engine, Plan 02's signature, and Plan 03's placed fields into the product's core deliverable.

Image-sourced documents (Phase 1 wraps images to a PDF on load) export uniformly: the wrapped PDF bytes are the "original" bytes, satisfying EXP-03 (PDF containing the image plus placed fields). The button is disabled until ≥1 field is placed, and any export failure surfaces an inline, dismissible error banner (T-02-02) without crashing.

Purpose: Make the signing loop deliver a real, downloadable, unaltered signed PDF — the single highest-intent action in the app.
Output: "Download PDF" in `TopBar.tsx`, `ExportErrorBanner.tsx`, original-bytes retention wired into `UploadZone.tsx` + `documentStore.ts`, the banner mounted in `App.tsx`, the download-wiring unit test, and end-of-phase human-verify checkpoints (full flow, EXP-03 visual, zero-third-party network audit).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| signature PNG data URL → pdf-lib embedPng | In-memory data URL (from canvas) crosses into the PDF decoder; could be malformed/oversized |
| original PDF bytes → pdf-lib PDFDocument.load | Untrusted uploaded file bytes parsed by the fork's PDF parser |
| npm install (pdf-lib-incremental-save fork) | Third-party low-usage package enters the dependency tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | exportSignedPdf data URL input | mitigate | Validate `dataUrl.startsWith('data:image/png;base64,')` before embedPng; throw tagged Error otherwise (V5 input validation) |
| T-02-02 | Denial of Service | exportSignedPdf on corrupt PDF | mitigate | Wrap entire export body in try/catch; re-throw tagged Error so caller surfaces ErrorBanner — never an unhandled rejection |
| T-02-03 | Information Disclosure | signature/PDF bytes | accept | All processing in-browser; no network calls in exportPdf/imageWrapper; PRV-01/PRV-02 architectural guarantee preserved |
| T-02-SC | Tampering | pdf-lib-incremental-save install (supply chain) | mitigate | Pin EXACT version 1.17.4; rely on lockfile; package audited OK in RESEARCH Package Legitimacy Audit (no postinstall, MIT). Verified [VERIFIED] in research → no human checkpoint required |
</threat_model>

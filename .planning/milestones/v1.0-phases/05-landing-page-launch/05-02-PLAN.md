## Phase Goal

**As a** privacy-conscious user about to sign a sensitive document, **I want to** be sure the site loads nothing from third parties and is deployment-ready, **so that** I can trust that my document and my visit are never observed by anyone.

<objective>
Lock in the zero-third-party-request posture with an automated source-scan guard, make the repo deployment-ready (vercel.json SPA rewrite + clean build), and document the human deploy runbook — then surface the deferred live deploy + network audit as an explicit human checkpoint.

This slice is demoable: run `npm test` and watch the privacy guard pass on the current source (and fail the moment an external `<script src>`/CDN font/`fetch` is added); run `npm run build` and get a self-contained `dist/`; read the README to know exactly how to ship it.

Purpose: PRV-03 (no third-party analytics/tracking/error-reporting — enforced by an automated regression guard) and LND-04 (deploy-ready: vercel.json + build + runbook; the actual deploy + DNS + live audit are deferred to the human).
Output: `src/test/privacyGuard.test.ts`; extended `vercel.json`; `README.md` deploy section; a blocking human-verify checkpoint for the deferred live deploy/audit.
</objective>

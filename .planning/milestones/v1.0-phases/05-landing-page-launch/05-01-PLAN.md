## Phase Goal

**As a** person who got fed up with fake-free PDF signers, **I want to** land on a page that candidly explains FreeESign is genuinely free and private and lets me jump straight into signing, **so that** I trust the tool enough to drop in my document.

<objective>
Build the public landing page and wire it as the new initial view, so booting the app shows a candid privacy-first landing that the user can click through into the existing signing tool.

This is the demoable slice: boot → see the landing page (hero + how-it-works + privacy + footer with a Buy Me a Coffee link) → click "Start signing" → arrive in the upload zone → click the TopBar wordmark → return to the landing page.

Purpose: LND-01 (candid hero), LND-02 (how-it-works + privacy statement), LND-03 (optional BMC support link), and the no-router view-machine wiring that makes the landing the app's front door.
Output: `'landing'` ViewState (initial) + `goToLanding()`/`startSigning()` actions; `src/config.ts`; `LandingPage.tsx` + 5 section components; App.tsx + TopBar.tsx wiring; self-hosted favicon + meta description; updated `documentStore.test.ts`; new `landingPage.test.tsx`.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | LandingFooter BMC link (`<a target="_blank">`) | mitigate | `rel="noopener noreferrer"` on the BMC anchor prevents reverse-tabnabbing (the opened tab cannot access `window.opener`). Enforced + tested in landingPage.test.tsx. |
| T-05-02 | Information Disclosure | LandingFooter / all landing components | mitigate | BMC is a plain navigation `<a>`, NEVER a `<script>`/widget/iframe (LND-03 / Area 3). landingPage.test.tsx asserts no `<script>`/`<iframe>` in the rendered landing; Plan 02 source-scan privacy guard backstops PRV-03. |
| T-05-03 | Tampering | index.html / favicon | mitigate | Favicon replaced with a self-hosted `public/favicon.svg` (no external href); no `<script src>`/`<link href>` to external origins added to `<head>`. |
| T-05-SC | Tampering | npm installs | accept | No new packages installed in this plan (RESEARCH: zero new deps). No package-legitimacy gate required. |

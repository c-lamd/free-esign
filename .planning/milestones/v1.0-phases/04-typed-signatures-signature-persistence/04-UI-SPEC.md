# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-crafted inline-style + Tailwind v4 components (same as Phase 1/2/3) |
| Icon library | none — inline SVG only (reuse × close from Phase 2/3; tab-underline indicator is CSS only) |
| Font | system-ui stack for UI chrome: `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |
| Script fonts (signature content ONLY) | Dancing Script, Great Vibes, Pacifico — self-hosted TTFs via `@font-face`; used exclusively to render signature/initials glyph content, never for UI chrome/labels |

**Rationale:** Phase 4 extends the established hand-crafted pattern from Phase 1/2/3. No new design system dependency. shadcn remains deferred to Phase 5.

**New libraries used (visual/interaction impact):**
- `idb-keyval@6.2.5` — IndexedDB key-value store for persistence; no visual impact.
- `@pdf-lib/fontkit@1.1.1` — font embedding in export layer; no visual impact.
- 3 self-hosted script TTFs (Dancing Script, Great Vibes, Pacifico) + OFL license files.

**Script font scope boundary (IMPORTANT for checker):**
The three script fonts are NOT part of the UI type scale. They are signature content assets — analogous to an image. The type scale (14/16/20/24px, weights 400/600, system-ui) governs all UI chrome: labels, headings, buttons, hint text. The script fonts are used only to render:
1. The live preview inside the Type tab (the rendered name glyph in the chosen font).
2. The on-canvas typed-signature widget inside `PlacedFieldWidget` (`fontFamily` CSS prop applied to the text content div).
3. The saved-item thumbnail text for typed items in the Saved tab.
A "type scale violation" checker flag for these three fonts is a false positive — they are content fonts, not UI typography.

## Spacing Scale

Inherited from Phase 1/2/3 unchanged. All tokens declared in `src/index.css` `:root` are reused exactly.

| Token | Value | Phase 4 Usage |
|-------|-------|--------------|
| xs | 4px | Tab bar gap, saved-item grid gap, font-option gap |
| sm | 8px | Tab button horizontal padding, font-option inner padding, saved-item thumbnail inner padding |
| md | 16px | Modal horizontal padding (unchanged), tab panel inner padding |
| lg | 24px | Modal header bottom margin, section rhythm within tab panels |
| xl | 32px | Modal vertical centering offset (unchanged) |
| 2xl | 48px | Modal vertical centering offset (unchanged) |
| 3xl | 64px | Not used in Phase 4 |

**Phase 4 exceptions:**
- Tab buttons in the tab bar: minimum 44px height (WCAG 2.5.5); visual height may be less with padding bringing touch target to 44px.
- Font-option cards: minimum 44px height touch target.
- Saved-item thumbnail cards: minimum 44px × 44px touch target (click to place).
- Save-for-reuse checkbox row: minimum 44px height touch target wrapping the checkbox + label.
- Delete per-item button: 32px × 32px visual, wrapped to 44px × 44px touch target via invisible padding (same pattern as Phase 2/3 delete control).
- **Intra-component 12px:** the Type-tab name input padding (`8px 12px`), font-picker `margin-top: 12px`, and live-preview box (`12px 16px` / `margin-top: 12px`) use 12px. 12px is a multiple of 4 (grid-aligned); it is an accepted intra-component value, not a new global token. Implementers may substitute `sm` (8px) or `md` (16px) if preferred — both are acceptable.

## Typography

Inherited from Phase 1/2/3 — the same four sizes and two weights apply. No new sizes or weights are introduced in Phase 4.

| Role | Size | Weight | Line Height | Phase 4 Usage |
|------|------|--------|-------------|--------------|
| Body | 16px | 400 (regular) | 1.5 | Type-tab name input placeholder, "no saved items" empty state body |
| Label | 14px | 400 (regular) | 1.4 | Tab bar labels (Saved/Draw/Type), font-option labels, "Save for reuse" checkbox label, saved-item caption text |
| Heading | 20px | 600 (semibold) | 1.2 | Modal title ("Create signature" / "Create initials") |
| Display | 24px | 600 (semibold) | 1.1 | "FreeESign" wordmark — unchanged |

Font weights declared: **400** and **600** only. No third weight is added in Phase 4.

**Name input:** rendered at Body size (16px/400) so the typed name is comfortable to read. Input font inherits system-ui stack.

**Script font usage (content only, not UI typography):**
- Live preview glyph: font-size scales to fill the preview container vertically (target ~70% of container height, using `clamp(24px, 10vw, 64px)` as a starting CSS value; executor adjusts to produce a visually balanced glyph fill). Font-family set to the selected font's CSS `@font-face` name.
- On-canvas widget text content: same `fontFamily` CSS prop as the selected font; `font-size` computed to approximately fill the widget's CSS height × 0.85 so it is WYSIWYG with the export. These sizes are NOT typography tokens — they are sizing heuristics for content layout.
- Saved-item thumbnail text: same font-family as the item's stored `fontFamily`; font-size 20px to fit the thumbnail preview area.

### SignatureDrawModal (extended from Phase 2 into tabbed modal)

The existing `SignatureDrawModal.tsx` is extended to a tabbed modal with three panels: **Saved · Draw · Type**. The modal shell (scrim, dialog dimensions, focus trap, Escape handler, focus restore on close) is identical to Phase 2.

**New: Tab bar**

Positioned between the modal title and the panel content area.

- Layout: `display: flex; flex-direction: row; border-bottom: 1px solid var(--color-border); margin-bottom: 16px`
- Tab button style (default/inactive): `background: none; border: none; padding: 8px 16px; fontSize: 14px; fontWeight: 400; color: var(--color-text-secondary); cursor: pointer; min-height: 44px; border-bottom: 2px solid transparent` (the 2px transparent bottom border reserves space so active tabs don't shift layout)
- Tab button style (active): `color: var(--color-text-primary); border-bottom: 2px solid var(--color-accent)`

- ARIA: `role="tablist"` on the container, `role="tab"` on each button, `aria-selected="true"` on the active tab, `aria-selected="false"` on others. Each tab has `aria-controls="panel-{saved|draw|type}"`. Each panel has `role="tabpanel"` and `id="panel-{saved|draw|type}"` and `aria-labelledby="{tab-id}"`.
- Keyboard: Left/Right arrow keys cycle between tabs (standard ARIA tab widget pattern). Tab key moves focus to the panel content, not to the next tab.

**Panel: Draw**

Identical to the Phase 2 modal content (signature_pad canvas, Clear canvas / Discard / Add signature controls). The "Add signature" button becomes "Use signature" when the flow produces a result (see Copywriting Contract — the button label is context-sensitive).

The "Save for reuse" checkbox row is appended below the controls row (see Save-for-reuse checkbox section below).

**Panel: Type**

New panel for typed signature creation.

Layout (top-to-bottom):
1. Name input — `<input type="text">` full-width
2. Font picker — 3 font-option cards in a row (or wrapping at narrow widths)
3. Live preview — a constrained-height preview box rendering the typed name in the selected font
4. "Save for reuse" checkbox row (shared with Draw panel)
5. Controls row: [Discard] [Use signature] — same ghost/accent button pattern as Draw panel

**Name input:**
- `width: 100%; padding: 8px 12px; font-size: 16px; font-weight: 400; font-family: inherit; color: var(--color-text-primary); background: var(--color-surface-elevated); border: 1px solid var(--color-border); border-radius: 6px; outline: none; box-sizing: border-box`

- `aria-label="Your name for signature"`

**Font picker:**

### PlacedFieldWidget (extended from Phase 3 — typed signature/initials branch)

Phase 3's `PlacedFieldWidget` gains a new render branch for font-backed fields: signature or initials fields where `dataUrl` is absent and `textValue` + `fontFamily` are present.

**Font-backed signature/initials widget:**
- Content: `<div style="width:100%;height:100%;display:flex;align-items:center;justify-content:center;overflow:hidden;font-family:{field.fontFamily};font-size:{computed}px;color:var(--color-text-primary);white-space:nowrap;user-select:none;pointer-events:none">{field.textValue}</div>`
- Font size computed at render: `Math.min(cssHeight * 0.85, cssWidth / (textValue.length * 0.6 + 0.5))` — a heuristic that fills height while preventing overflow on long names. Executor may refine but must not exceed the widget bounds.
- `lockAspectRatio={false}` — typed signatures can be resized in both dimensions freely (unlike drawn PNG which has a natural aspect ratio). The user controls proportions.
- Default placed size: width 200px, height 56px (CSS pixels at current scale) — wider-than-tall default suits a typed name. Coordinate Mapper converts to PDF space immediately on placement.
- Delete control: same as drawn signature — "Delete signature" / "Delete initials" per type.
- `role="img"` + `aria-label="Placed typed signature — press Delete to remove"` / `"Placed typed initials — press Delete to remove"` on the widget wrapper.

**Image-backed branch (drawn signature/initials):**
- Unchanged from Phase 2/3: `<img src={field.dataUrl} ... />` with `lockAspectRatio={true}`.

## Self-Hosted Font Assets (PRV-02)

Phase 4 introduces script font files. These must be self-hosted — no CDN, no Google Fonts URL.

**File layout:**
```
public/
  fonts/
    DancingScript-Regular.ttf
    DancingScript-LICENSE.txt      (OFL)
    GreatVibes-Regular.ttf
    GreatVibes-LICENSE.txt         (OFL)
    Pacifico-Regular.ttf
    Pacifico-LICENSE.txt           (OFL)
```

**`@font-face` declarations** (added to `src/index.css`):
```css
@font-face {
  font-family: "Dancing Script";
  src: url("/fonts/DancingScript-Regular.ttf") format("truetype");
  font-weight: 400;
  font-style: normal;
  font-display: block;
}

@font-face {
  font-family: "Great Vibes";
  src: url("/fonts/GreatVibes-Regular.ttf") format("truetype");
  font-weight: 400;
  font-style: normal;
  font-display: block;
}

@font-face {
  font-family: "Pacifico";
  src: url("/fonts/Pacifico-Regular.ttf") format("truetype");
  font-weight: 400;
  font-style: normal;
  font-display: block;
}
```

`font-display: block` is used (not `swap`) to prevent a flash of unstyled system text in the live preview and on-canvas widget — for a signature element, an unstyled flash is especially visually jarring.

**Font-loading for export (pdf-lib/fontkit):**
Load the same TTF bytes used for on-screen display to embed in the PDF:
```ts
const fontBytes = await fetch('/fonts/DancingScript-Regular.ttf').then(r => r.ArrayBuffer())
```
This is a same-origin request to the Vite/Vercel static asset origin — NOT a third-party request. PRV-02 is preserved.

**Font CSS name → file mapping (executor reference):**
| CSS family name | File |
|----------------|------|
| `"Dancing Script"` | `DancingScript-Regular.ttf` |
| `"Great Vibes"` | `GreatVibes-Regular.ttf` |
| `"Pacifico"` | `Pacifico-Regular.ttf` |

## Accessibility Requirements

All Phase 1, 2, and 3 requirements carry forward. Phase 4 additions:

- **Tab widget:** `role="tablist"` on the container, `role="tab"` on each button, `aria-selected` on each tab, `role="tabpanel"` on each panel with `aria-labelledby`. Keyboard: Left/Right arrow keys cycle tabs (standard ARIA tabs pattern); Tab key exits the tab bar and moves focus into the panel content.
- **Font picker radiogroup:** `role="radiogroup"` with `aria-label="Choose script font"`. Each option card has `role="radio"` and `aria-checked`. Space/Enter activates the focused option. Arrow keys navigate between options.
- **Saved-item radiogroup:** `role="radiogroup"` with `aria-label`. Each `SavedItemCard` has `role="radio"` and `aria-checked`. Arrow keys (Left/Right/Up/Down grid-navigation) move between cards. Enter or Space selects the card (and arms it if the "Use" CTA is not available). The delete button inside the card is reachable via Tab from within the card (not via radiogroup arrow navigation — it is a separate focusable element).
- **Live preview aria-live:** `aria-live="polite"` on the preview container so screen readers announce when the preview content changes (font or text changes).
- **Save-for-reuse checkbox:** properly associated via `<label htmlFor>` — no extra aria-label needed (label text is the accessible name).
- **"Use" CTA disabled states:** `aria-disabled="true"` (not HTML `disabled`) in all three panels when no content is ready — same aria-disabled pattern as all previous phases. Each has a descriptive `aria-label` for the disabled state (see Copywriting Contract).
- **SavedItemCard delete button:** `aria-label` + `<span class="sr-only">` dual-label pattern (same as all phase 2/3 delete controls). `stopPropagation` so activating the delete does not also trigger card selection.
- **Focus trap:** The tabbed modal maintains the same focus trap as Phase 2 SignatureDrawModal. Tab cycles within the modal. Escape closes and returns focus to the trigger element.
- **Focus order within modal:** Title → Tab bar (tab key to first panel element) → panel content → Save-for-reuse → CTA buttons.
- **Font loading for preview:** `font-display: block` prevents FOUT (flash of unstyled text) — critical for a signature preview where a brief system-font flash would appear as garbled content.
- **Color contrast (new pairs):**
  - Active tab label `--color-text-primary` (#111827) on white: 16.75:1 — passes AA
  - Inactive tab label `--color-text-secondary` (#6B7280) on white: 4.63:1 — passes AA
  - Typed signature text `--color-text-primary` on `--color-surface` (#F9FAFB): 15.7:1 — passes AA
  - SavedItemCard "Drawn"/"Typed" label `--color-text-secondary` on `--color-surface`: 4.63:1 — passes AA
  - White × on `--color-destructive` (#DC2626) for the 20px delete dot: 4.47:1 — passes AA (large icon)

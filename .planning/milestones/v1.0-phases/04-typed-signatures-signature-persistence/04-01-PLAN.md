<objective>
Lay the Phase 4 foundation slice: install the two new build-time dependencies, vendor the three self-hosted script fonts, and extend the field store + persistence layer so saved signatures/initials can be stored in IndexedDB and reused across sessions. Also extend the field model with the `fontFamily` field and the `armedTypedPayload` seam that later plans consume to place typed signatures.

This slice is demoable at the store level: after it lands, `addSavedItem` writes an item to IndexedDB and surfaces it in `savedItems`, `loadSavedItems` hydrates from IndexedDB, and `deleteSavedItem` removes it — all proven by `savedItems.test.ts` with a mocked `idb-keyval`. The fonts are on disk and declared via `@font-face` so the on-screen and export layers (Plans 02/03) can render WYSIWYG.

Purpose: every other Phase 4 plan depends on the store shape, the persistence helpers, the installed deps, and the vendored fonts. Building them first unblocks Waves 2 and 3.
Output: installed deps, `public/fonts/` assets, `@font-face` CSS, extended `fieldStore.ts`, new `savedSignatures.ts`, new `savedItems.test.ts`, and idb-keyval mock added to `fieldStore.test.ts`.

Implements SIG-04 (persistence across sessions) and SIG-05 (view/delete) at the data layer; establishes the `fontFamily` + `armedTypedPayload` seam for SIG-02/SIG-03 (consumed by Plans 02/03).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → build | new packages (@pdf-lib/fontkit, idb-keyval) enter the supply chain |
| user → IndexedDB | saved-item payloads (typed text, drawn PNG dataURLs) are written to origin-local IndexedDB |
| static origin → app | TTF files served from public/fonts/ (same-origin) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-SC | Tampering | npm install of @pdf-lib/fontkit + idb-keyval | mitigate | Task 1 blocking-human legitimacy checkpoint for idb-keyval (SUS/too-new); fontkit is OK/Approved; both pinned to exact versions in package.json |
| T-04-01 | Tampering | SavedItem.fontFamily stored in IndexedDB | mitigate | Store only the fontFamily string (allowlist of 3 names), never font bytes; bytes always loaded from the static public/fonts/ allowlist (enforced in Plan 02 fonts.ts) |
| T-04-02 | Information disclosure | idb-keyval persistence | accept | IndexedDB is origin-local and unprivileged; no PII leaves the browser; consistent with the privacy-first, no-server model |
| T-04-03 | Tampering | malicious data in IndexedDB on restore | mitigate | SavedItem stores text/dataUrl/fontFamily only — no executable content; React (Plan 03) renders text as text nodes, never dangerouslySetInnerHTML; PNG dataUrls validated at export time (existing exportPdf PNG_DATA_URL_PREFIX guard) |
</threat_model>

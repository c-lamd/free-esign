Zustand store (src/store/documentStore.ts) — the view state machine:
```typescript
export type ViewState = 'empty' | 'loading' | 'error' | 'loaded'

export interface DocumentStore {
  view: ViewState
  docUrl: string | null        // Blob URL of current PDF (or wrapped image)
  numPages: number | null
  currentPage: number          // 1-indexed
  errorMessage: string | null

  setView: (v: ViewState) => void
  loadDocument: (url: string) => void   // sets docUrl, view='loading', clears error
  setNumPages: (n: number) => void      // sets numPages, view='loaded'
  setCurrentPage: (n: number) => void
  setError: (msg: string) => void       // sets errorMessage, view='error'
  reset: () => void                     // back to empty, revoke handled by caller
}
```

pdf worker module (src/lib/pdfWorker.ts):
```typescript
// Importing this module sets pdfjs.GlobalWorkerOptions.workerSrc = '/pdf.worker.min.mjs'
export const pdfOptions: {
  cMapUrl: '/cmaps/'
  cMapPacked: true
  standardFontDataUrl: '/standard_fonts/'
}
```

## Phase Goal

**As a** person who needs to sign a document, **I want to** open a PDF or image and see every page rendered in my browser, **so that** I can review it knowing the file never leaves my device.

(This plan delivers the thinnest end-to-end slice of that goal: pick a PDF, see page 1 render locally. Plans 02-04 complete the full upload UX, image support, and multi-page navigation.)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user file → browser memory | An untrusted PDF/image file enters the app via the file picker. |
| app → network | The only legitimate network egress is to the app's own origin for assets; any third-party request is a privacy violation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | pdfjs-dist CMap/worker/font loading | mitigate | Self-host worker (public/pdf.worker.min.mjs) + cmaps + standard_fonts; pass `options={pdfOptions}` to <Document> so pdfjs never falls back to a CDN (RESEARCH Pitfall 2). Verified by the Task 4 DevTools Network audit. |
| T-01-02 | Tampering / EoP | malicious PDF in pdfjs renderer | accept (with control) | pdfjs-dist sandboxes rendering in a Web Worker; `renderAnnotationLayer={false}` removes the annotation-injection surface. Full size/type ingestion validation is added in Plan 03. |
| T-01-03 | Tampering | pdf.js annotation-layer injection | mitigate | `renderAnnotationLayer={false}` on every <Page> (RESEARCH Security Domain). |
| T-01-SC | Tampering | npm installs (scaffold) | mitigate | RESEARCH "Package Legitimacy Audit" cleared all packages (no SLOP; SUS verdicts are `too-new`-only on official, high-download packages). No blocking checkpoint required. |
</threat_model>

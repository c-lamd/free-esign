<objective>
Build the Coordinate Mapper: a pure TypeScript module that converts between CSS pixel space (top-left origin, as rendered by react-pdf) and PDF user space (bottom-left origin, points at 72 DPI) across any zoom scale and page rotation. It is a thin, fully-testable wrapper over a pdfjs `PageViewport`'s `convertToPdfPoint` / `convertToViewportPoint`.

This is the explicit hand-off contract to Phase 2 field placement (success criterion 5) and underpins correct page geometry for the multi-page viewer (DOC-03). It ships with a round-trip property test that is a Phase 1 success criterion.

Purpose: Give Phase 2 a single, proven coordinate primitive so field placement math is never hand-rolled. Isolating it as a pure module makes the round-trip property unit-testable without a browser.
Output: src/lib/coordinateMapper.ts and a passing round-trip property test.
</objective>

```typescript
export interface PageSpace { x: number; y: number }   // PDF user space, bottom-left origin, points
export interface CssSpace  { x: number; y: number }   // CSS pixels, top-left origin, from rendered canvas

// viewport = pdfPage.getViewport({ scale, rotation })
export function cssPixelToPageSpace(css: CssSpace, viewport: { convertToPdfPoint(x: number, y: number): number[] }): PageSpace
export function pageSpaceToCssPixel(pdf: PageSpace, viewport: { convertToViewportPoint(x: number, y: number): number[] }): CssSpace
```
</interfaces>

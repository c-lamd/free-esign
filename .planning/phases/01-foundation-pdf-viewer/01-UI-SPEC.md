# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-crafted Tailwind v4 components |
| Icon library | none in Phase 1 — use inline SVG for the single upload icon and prev/next chevrons |
| Font | system-ui stack — `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |

**Rationale:** CLAUDE.md specifies Tailwind v4 (CSS-native config, no tailwind.config.js) as the styling layer with no shadcn reference. Phase 1 is a minimal tool surface (upload state + viewer + top bar) — hand-crafted components with Tailwind utilities are sufficient and avoid introducing a design-system dependency before the visual identity is finalized (brand is deferred to Phase 5). shadcn can be re-evaluated at Phase 5 landing-page time.

## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 default spacing scale is used; custom aliases below name the values that matter for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge padding |
| sm | 8px | Button horizontal padding (compact), input inner padding |
| md | 16px | Default element spacing, card inner padding, top-bar horizontal padding |
| lg | 24px | Section vertical rhythm, upload zone inner padding |
| xl | 32px | Page canvas side margin at desktop, major vertical gaps |
| 2xl | 48px | Upload zone vertical centering offset |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Touch targets for prev/next buttons and the "browse" button: minimum 44px × 44px (accessibility — WCAG 2.5.5 target size). The visual size may be smaller; add invisible padding to reach 44px.
- The drag-drop zone uses `min-height: 100dvh` minus the top-bar height (56px), not a fixed spacing token.

## Typography

All sizes are in px at base `font-size: 16px` (browser default, do not override). Tailwind v4 utility classes map 1:1.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Upload zone privacy line, error messages, page count label |
| Label | 14px | 400 (regular) | 1.4 | "Browse" button label, top-bar document actions, prev/next button labels |
| Heading | 20px | 600 (semibold) | 1.2 | Upload zone prompt ("Drop your document here"), error headings |
| Display | 24px | 600 (semibold) | 1.1 | "FreeESign" wordmark in the top bar |

Font weights declared: 400 (regular) and 600 (semibold). No exceptions. If additional wordmark presence is needed, use `letter-spacing`, not a third weight.

## Color

Tailwind v4 CSS custom properties are the mechanism. Declare these in `src/index.css` under `:root`.

| Role | Value | CSS Token | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | `#F9FAFB` (gray-50) | `--color-surface` | App background, page canvas backdrop base |
| Secondary (30%) | `#FFFFFF` (white) | `--color-surface-elevated` | Top bar background, rendered PDF page background, error card surface |
| Canvas backdrop | `#E5E7EB` (gray-200) | `--color-canvas` | Neutral gray behind rendered PDF pages (from CONTEXT.md decision) |
| Accent (10%) | `#2563EB` (blue-600) | `--color-accent` | See reserved list below |
| Destructive | `#DC2626` (red-600) | `--color-destructive` | Inline file error state icon and border only |
| Text primary | `#111827` (gray-900) | `--color-text-primary` | Body text, headings, wordmark |
| Text secondary | `#6B7280` (gray-500) | `--color-text-secondary` | Privacy line, supporting labels, page count |
| Border | `#D1D5DB` (gray-300) | `--color-border` | Top bar bottom border, upload zone dashed border, error card border |

**Accent (`#2563EB`) reserved for exactly:**
1. The "browse" button background (primary interactive CTA in the upload state)
2. The drag-over state border of the upload zone (4px solid accent ring)
3. The prev/next navigation button focus ring (2px outline)
4. Spinner stroke color

Accent is NOT used for: body text, labels, borders in non-interactive state, the "FreeESign" wordmark, the top bar background, or page navigation text.

**Destructive (`#DC2626`) reserved for exactly:**
1. The error card left border (4px) when a file fails validation or parse
2. The error icon (inline SVG, 16px)

## Component Inventory

Components to build in Phase 1 (no shadcn — all hand-crafted with Tailwind v4):

### TopBar
- Height: 56px fixed
- Background: `--color-surface-elevated` (white)
- Bottom border: 1px solid `--color-border`
- Left slot: "FreeESign" wordmark — Display size (24px/600)
- Right slot: "Open another" button — shown only when a document is loaded; Label size (14px/400), text style, accent color on hover
- Horizontal padding: 16px (md)

### UploadZone (empty state)
- Fills the full viewport height minus TopBar (min-height: calc(100dvh - 56px))
- Background: `--color-surface` (gray-50)
- Center-aligned content column, max-width 480px
- Upload icon: inline SVG, 48px × 48px, `--color-text-secondary`
- Heading: "Drop your document here" — Heading size (20px/600), `--color-text-primary`
- Subtext: "or" — Body size (16px/400), `--color-text-secondary`
- Browse button: accent background (#2563EB), white text (14px/400), 8px vertical padding, 16px horizontal padding, 6px border-radius, min 44px height
- Privacy line: "Your files never leave your browser." — Body size (16px/400), `--color-text-secondary`, 24px margin-top
- Dashed border on drag-over: 2px dashed `--color-accent`, 8px border-radius, covers the entire zone
- Accepts: `.pdf,.jpg,.jpeg,.png`

### LoadingSpinner
- Centered in the viewer area (not full-screen overlay)
- Size: 32px × 32px SVG spinner
- Stroke: `--color-accent`
- Shown while a document is parsing/rendering; hidden once first page renders

### ErrorBanner (inline, not modal)
- Replaces the UploadZone content (not a toast)
- Left border: 4px solid `--color-destructive`
- Background: white, border-radius 6px, padding 16px
- Icon: 16px red warning SVG
- Heading: "Could not open file" — Heading size (20px/600), `--color-text-primary`
- Body: dynamic error copy (see Copywriting Contract below) — Body size (16px/400), `--color-text-primary`
- Retry action: "Try another file" — Label size (14px/400), `--color-accent`, underline on hover

### DocumentViewer
- Background: `--color-canvas` (gray-200)
- Pages centered horizontally, max-width = fit-to-width of the viewport minus scrollbar width
- Each page: white background, rendered by react-pdf `<Page>` component, with a subtle box shadow: `0 2px 8px rgba(0,0,0,0.12)`
- Page gap: 24px (lg) between successive pages
- Vertical padding: 24px top and bottom within the canvas

### PageNavigation
- Fixed position: bottom-center of the viewport, 24px above the bottom edge
- Background: white, border-radius 20px (pill), box-shadow: `0 2px 8px rgba(0,0,0,0.15)`, padding: 8px 16px
- Layout: [prev button] [page indicator] [next button], gap 8px
- Prev/Next: inline SVG chevron (16px), min 44px touch target, `--color-text-primary`, disabled state at opacity 0.35 when at first/last page. Each button must include both `aria-label="Previous page"` / `aria-label="Next page"` AND a visually-hidden `<span class="sr-only">Previous page</span>` / `<span class="sr-only">Next page</span>` inside the button element so a visible-in-DOM label exists for keyboard users.
- Page indicator: "1 / N" — Label size (14px/400), `--color-text-secondary`

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (upload empty state) | "Browse files" |
| Empty state heading | "Drop your document here" |
| Empty state body (privacy line) | "Your files never leave your browser." |
| Secondary prompt | "or" (between heading and browse button) |
| Loading state | No text — spinner only |
| Error heading | "Could not open file" |
| Error: corrupt / unreadable file | "This file couldn't be read. It may be corrupt or password-protected. Try another file." |
| Error: wrong file type | "Only PDF, JPG, and PNG files are supported. Try another file." |
| Error: file too large (>100MB) | "This file is too large to open in the browser. Try a smaller file." |
| Retry action label | "Try another file" |

| Page indicator | "1 / N" (N = total page count) |

| Next button aria-label | "Next page" |

**Voice notes:** Copy is calm, direct, and non-alarmist. Errors name the problem and always give the user a next step. The privacy line is a statement of fact, not marketing language — no exclamation marks.

## Layout Architecture

```
┌─────────────────────────────────────────────────┐
│  TopBar (56px, position: sticky, top: 0)         │
│  [FreeESign wordmark]         [Open another]     │
├─────────────────────────────────────────────────┤
│                                                   │
│  UploadZone OR DocumentViewer                     │
│  (min-height: calc(100dvh - 56px))               │
│                                                   │
│  Empty:    UploadZone (centered content column)   │
│  Loading:  LoadingSpinner (centered in canvas)    │
│  Error:    ErrorBanner (centered in canvas)       │
│  Loaded:   DocumentViewer (scrollable)            │
│            ┌─────────────────────────────┐        │
│            │   [Page 1 — white bg]       │        │
│            │   (shadow, centered)        │        │
│            ├─────────────────────────────┤        │
│            │   24px gap                  │        │
│            ├─────────────────────────────┤        │
│            │   [Page 2 — white bg]       │        │
│            └─────────────────────────────┘        │
│                                                   │
│         ┌──────────────────────┐                  │
│         │ ‹  1 / 3  ›         │  ← PageNav pill  │
│         └──────────────────────┘                  │
└─────────────────────────────────────────────────┘
```

**State machine (Zustand):** `view` ∈ `{ empty | loading | error | loaded }`

Only one of `UploadZone / LoadingSpinner / ErrorBanner / DocumentViewer` renders at a time.

## Accessibility Requirements

- All interactive elements have visible focus rings (2px outline, `--color-accent`, 2px offset)
- Touch targets: minimum 44px × 44px for all buttons (add invisible padding where needed)
- The drag-drop zone has `role="region"` and `aria-label="Document upload area"`
- The file input (`<input type="file">`) is visually hidden but accessible — the Browse button activates it
- Page indicator is `aria-live="polite"` so screen readers announce page changes
- Prev/next buttons use `aria-disabled="true"` (not `disabled`) at boundaries so focus remains reachable
- Prev/next buttons are icon-only visually but must include both `aria-label` and a `<span class="sr-only">` text label inside the button — the `sr-only` class uses `position: absolute; width: 1px; height: 1px; overflow: hidden; clip: rect(0,0,0,0); white-space: nowrap` and must be defined in `src/index.css`
- Error banner has `role="alert"` so screen readers announce it immediately on appearance
- Color contrast: all text/background combinations must meet WCAG AA (4.5:1 for body text, 3:1 for large text)
  - `--color-text-primary` (#111827) on white: 16.75:1 — passes
  - `--color-text-secondary` (#6B7280) on `--color-surface` (#F9FAFB): 4.63:1 — passes AA
  - White text on `--color-accent` (#2563EB): 4.54:1 — passes AA

## Self-Hosted Asset Requirements (PRV-02)

These are a Phase 1 success criterion, not a nice-to-have. The executor must:

1. Configure Vite to copy the pdfjs-dist worker file to `public/` and set `GlobalWorkerOptions.workerSrc` to the local path
2. Download the required CMaps from pdfjs-dist and bundle them under `public/cmaps/`
3. No external font requests — system-ui stack only in Phase 1 (script fonts are a Phase 4 concern)
4. Verify with DevTools Network tab: after loading a document, zero requests to third-party origins

These constraints are non-visual but are part of the design contract because they govern what assets the UI may reference.

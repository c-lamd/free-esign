<objective>
Add user-controlled document zoom (50–200%, discrete steps) with fields that scale and stay pixel-aligned — closing DOC-04 — by threading a single `effectiveScale = (containerWidth / originalWidth) * zoom` through both the react-pdf page render width and the field-overlay viewport, while NEVER mutating the PDF-space coordinates stored on fields.

This is the workspace-zoom vertical slice: a user opens a document, presses + / − on a floating pill near the page-nav, and the page and every placed field grow/shrink together, staying locked to the same spot on the content. The provability guarantee (the whole point of zoom per CONTEXT.md "Specific Ideas") is captured by a coordinate test asserting the same document point maps to identical PDF coordinates regardless of zoom.

Per CONTEXT.md Area 2 (range/steps, scale model, field alignment, control placement, anchoring) and RESEARCH Section 3 (react-pdf width+scale interaction, effectiveScale threading, ZOOM_STEPS, page-anchor on zoom).

Purpose: Zoom is essential for precise placement on dense documents; the zero-drift guarantee is what makes placement trustworthy.
Output: `documentStore.ts` zoom state + ZOOM_STEPS; new `ZoomControl.tsx`; `LazyPage.tsx` effectiveScale threading; `DocumentViewer.tsx` renders ZoomControl; zoom-step + zoom-invariance tests.
</objective>

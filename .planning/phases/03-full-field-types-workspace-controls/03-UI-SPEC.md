# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-crafted inline-style + Tailwind v4 components (same as Phase 1/2) |
| Icon library | none — inline SVG only (new: undo/redo arrow SVGs, zoom +/- SVGs; reuse × close from Phase 2) |
| Font | system-ui stack — `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` |

**Rationale:** Phase 3 extends Phase 1/2's established hand-crafted pattern. No new design system dependency is introduced. shadcn remains deferred to Phase 5.

**New libraries used (visual/interaction impact):**
- No new libraries. All new UI is built with existing stack (React, react-rnd, Tailwind v4, Zustand).

## Spacing Scale

Inherited from Phase 1/2 unchanged. All tokens declared in `src/index.css` `:root` are reused exactly.

| Token | Value | Phase 3 Usage |
|-------|-------|--------------|
| xs | 4px | Gap between field-palette buttons, zoom control inner gap |
| sm | 8px | Palette button padding, zoom button padding, undo/redo button padding |
| md | 16px | TopBar horizontal padding (unchanged), banner padding |
| lg | 24px | Bottom chrome clearance, section rhythm |
| xl | 32px | Not used in new Phase 3 components |
| 2xl | 48px | Not used in new Phase 3 components |
| 3xl | 64px | Not used in Phase 3 |

**Phase 3 exceptions:**
- Field palette buttons: minimum 44px height (WCAG 2.5.5); visual height may be less with padding bringing touch target to 44px.
- Zoom +/- buttons: minimum 44px × 44px touch target.
- Undo/Redo buttons: minimum 44px × 44px touch target.
- Initials draw modal: same constraints as Phase 2 SignatureDrawModal (max-width 560px, width calc(100vw - 32px)).
- Inline text-field input: minimum height 24px (matches PlacedFieldWidget minHeight); no artificial minimum forced beyond the widget resize floor.

## Typography

Inherited from Phase 1/2 — the same four sizes and two weights apply. No new sizes or weights are introduced in Phase 3.

| Role | Size | Weight | Line Height | Phase 3 Usage |
|------|------|--------|-------------|--------------|
| Body | 16px | 400 (regular) | 1.5 | Word-doc rejection banner body, placement-mode banner update |
| Label | 14px | 400 (regular) | 1.4 | Field palette button labels, zoom percentage readout, undo/redo button labels, inline text/date field inputs |
| Heading | 20px | 600 (semibold) | 1.2 | Initials draw modal title ("Draw your initials") |
| Display | 24px | 600 (semibold) | 1.1 | "FreeESign" wordmark — unchanged from Phase 1 |

Font weights declared: **400** and **600** only. No third weight is added in Phase 3.

**Inline text/date field inputs:** rendered at Label size (14px/400) so the typed content fits comfortably at the widget's minimum height. Input font inherits system-ui stack.

**Rendered PDF text (export layer):** Helvetica (PDF StandardFonts) for text and date fields. Checkbox/X mark: Helvetica-Bold drawn via `drawText('✕', ...)` — a bold X character, not drawn lines, for crispness. These are PDF rendering concerns, not UI typography tokens.

2. Drag-over state border of the upload zone (Phase 1 — unchanged)
3. Prev/next navigation button focus ring (Phase 1 — unchanged)
4. Spinner stroke color (Phase 1 — unchanged)

### FieldPalette (new — inside TopBar)

A compact group of labeled buttons placed in the TopBar right slot, before "Download PDF".

**Layout:**
- Positioned in TopBar right slot in this order (left to right): `[Signature] [Initials] [Date] [Text] [Checkbox] | [Download PDF] [Open another]`
- Visual separator between `[Checkbox]` and `[Download PDF]`: `1px solid var(--color-border)`, height 20px, `margin: 0 4px` — a thin vertical rule, not a button
- Button style (unarmed/default): ghost — no background, `--color-text-secondary` label (14px/400), no border, 8px vertical padding, 8px horizontal padding, 6px border-radius, min 44px height. Same ghost style as Phase 2 "Add signature" button.
- Button style (armed/active): accent background (#2563EB), white label (14px/400), 6px border-radius, min 44px height. Only one button is armed at a time.
- Button style (hover, unarmed): label shifts to `--color-text-primary`; no background fill
- Button style (hover, armed): background shifts to `#1D4ED8` (blue-700) — same hover pattern as primary buttons
- Button style (focus, any state): 2px outline `--color-accent`, 2px offset
- Labels: "Signature", "Initials", "Date", "Text", "Checkbox" — exact strings, sentence case
- `aria-pressed="true"` on the currently armed button; `aria-pressed="false"` on all others. This communicates toggle state to screen readers without requiring a role="toolbar".
- `aria-label` on each: "Place signature field", "Place initials field", "Place date field", "Place text field", "Place checkbox field"

**Armed state behavior (visual only — interaction logic is in executor scope):**
- Exactly one or zero buttons is armed at a time. Clicking an armed button disarms it (toggling). Clicking another button arms it and disarms the previous.
- When armed, the PlacementModeOverlay banner appears with updated copy for the armed type (see Copywriting Contract).
- After drop: the button disarms automatically.

Two ghost buttons placed in the TopBar right slot at the leftmost position (before the FieldPalette group).

**Layout:**

- Hover (enabled): icon shifts to `--color-text-primary`
- Focus: 2px outline `--color-accent`, 2px offset
- Icon: inline SVG — 16px × 16px. Undo: counterclockwise curved arrow. Redo: clockwise curved arrow.

**Full TopBar right-slot order (document loaded):**
`[Undo] [Redo] | [Signature] [Initials] [Date] [Text] [Checkbox] | [Download PDF] [Open another]`

### ZoomControl (new — near PageNavigation pill)

A floating pill adjacent to the existing PageNavigation pill at the bottom-center of the viewport.

**Layout:**
- Position: `position: fixed; bottom: 24px; left: 50%` — but offset left of the PageNavigation pill so both are visible without overlap. The two pills are placed as a row centered at bottom: the ZoomControl appears to the left of the PageNavigation pill with a 12px gap. Total row: `[ZoomControl pill] [12px gap] [PageNavigation pill]`, centered horizontally via a wrapper div.
- Pill shape: same as PageNavigation — `background: var(--color-surface-elevated); border-radius: 20px; box-shadow: 0 2px 8px rgba(0,0,0,0.15); padding: 8px 16px; display: flex; align-items: center; gap: 8px; z-index: 50`
- Contents (left to right): `[zoom-out button] [percentage readout] [zoom-in button] [fit-width button]`
- Zoom-out button: inline SVG minus icon (16px), 44px × 44px touch target, ghost style matching PageNav prev/next
- Zoom percentage readout: Label size (14px/400), `--color-text-secondary`, min-width 44px, text-align center, `aria-live="polite"` so screen readers announce zoom changes
- Zoom-in button: inline SVG plus icon (16px), 44px × 44px touch target, ghost style
- Fit-width button: "Fit" label (14px/400), `--color-text-secondary`, ghost style, 44px touch target. Resets zoom to 1.0 (fit-to-width).
- Disabled state for zoom-out at 50% and zoom-in at 200%: opacity 0.35, `aria-disabled="true"`, cursor default. Same aria-disabled pattern as Phase 1/2.

**Zoom steps:** 50%, 75%, 100%, 125%, 150%, 175%, 200%. Each button press advances one discrete step.

### PlacedFieldWidget (extended from Phase 2)

Phase 2 PlacedFieldWidget is extended to render four additional field types. The react-rnd wrapper, selection chrome (border, handles, × delete), drag/resize behavior, and PDF-space coordinate storage all remain identical.

**Per-type visual content (inside the react-rnd bounds):**

**Signature / Initials (image types):**
- Unchanged: `<img src={field.dataUrl} ... objectFit: 'contain'>`
- Initials have `lockAspectRatio={true}` (same as signature). Initials default placed size: 80px wide (CSS pixels) × calculated height from PNG aspect ratio. Minimum resize: same as signature (80px × 24px).
- Delete control `aria-label`: "Delete initials" for initials type; "Delete signature" for signature type.

**Date field:**
- Content: a single-line `<input type="text">` filling the widget bounds.
- Input style: `width: 100%; height: 100%; border: 1px solid var(--color-border); border-radius: 3px; padding: 0 4px; font-size: 14px; font-weight: 400; font-family: inherit; color: var(--color-text-primary); background: transparent; box-sizing: border-box; outline: none`
- Focus state: `border-color: var(--color-accent)` (via onFocus/onBlur handlers, same inline-style pattern)
- Default value: today's date in `M/D/YYYY` format (e.g., "6/17/2026"), computed at field creation via `new Date()`. The user may edit the literal string freely.
- `lockAspectRatio={false}` — date fields can be resized freely in both dimensions.
- `minWidth={80}` `minHeight={24}` — same resize floor as signature.
- Delete control `aria-label`: "Delete date field".
- `aria-label` on input: "Date field value"
- Keyboard: when focus is inside the `<input>`, Delete/Backspace must NOT fire the field-deletion keyboard handler. Guard: check `document.activeElement.tagName === 'INPUT'` before triggering deletion and before triggering Ctrl+Z / Ctrl+Y undo/redo shortcuts.

**Text field:**
- Content: a single-line `<input type="text">` filling the widget bounds, identical style to the date input.
- Placeholder: "Type here" (shown in `--color-text-secondary` via `::placeholder` — add to index.css: `input::placeholder { color: var(--color-text-secondary); }`)
- Default value: empty string.
- `lockAspectRatio={false}` — text fields can be resized freely in both dimensions.
- `minWidth={80}` `minHeight={24}` — same resize floor.
- Delete control `aria-label`: "Delete text field".
- `aria-label` on input: "Text field content"
- Keyboard: same guard as date field — no undo/delete shortcuts while input is focused.
- Single-line for v1; `overflow: hidden; white-space: nowrap` on the input wrapper.

**Checkbox/X field:**
- Content: a `<div>` filling the widget bounds, centered, displaying a bold "✕" character.
- Renders as: `display: flex; align-items: center; justify-content: center; font-size: min(field CSS height * 0.7, field CSS width * 0.7)px; font-weight: 700; color: var(--color-text-primary); user-select: none; pointer-events: none`
- The font-size scales with the widget size (responsive to resize). Use `Math.min(cssHeight * 0.7, cssWidth * 0.7)` computed inline.
- `lockAspectRatio={true}` — checkboxes should stay square-ish; aspect lock is appropriate.
- Default placed size: 32px × 32px (CSS pixels at current scale). Minimum resize: 20px × 20px.
- Delete control `aria-label`: "Delete checkbox field".
- `role="img"` and `aria-label="Placed checkbox mark — press Delete to remove"` on the widget outer wrapper.
- Not interactive (no click to toggle) — placing it = marking it. The presence of the ✕ character is the mark.

**Selected-state chrome (all types):**
- Border: `2px solid var(--color-accent)` — unchanged
- Resize handle dots: 8px accent circles — unchanged
- × delete button at top-right: unchanged position and style. Per-type `aria-label` as specified above.

### InitialsDrawModal (new — mirrors SignatureDrawModal)

A centered, focus-trapped modal dialog for drawing initials. Structure is identical to Phase 2 SignatureDrawModal with the following differences:

- Modal `aria-labelledby` title: "Draw your initials"
- Canvas hint text (empty state): "Draw here"
- Primary CTA label: "Add initials"
- `aria-disabled` label when canvas empty: `aria-label="Add initials — draw initials first"`
- Cancel button label: "Discard" (same as SignatureDrawModal)
- Clear button label: "Clear canvas" (same as SignatureDrawModal)
- After confirm: the initials PNG data URL is used to arm an "initials" placement (not "signature"). Placement mode banner shows the initials-specific copy (see Copywriting Contract).

- Canvas aspect ratio: 2:1 (wider than tall suits initials — e.g., "JD" is wider than tall). Use `padding-top: 50%` for the aspect-ratio trick. All other layout metrics match SignatureDrawModal.
- Focus management: identical to SignatureDrawModal.

**PlacedField model extension:**
```
{
  id: string            // uuid
  type: 'signature' | 'initials' | 'date' | 'text' | 'checkbox'
  pageNumber: number    // 1-indexed
  pdfX: number
  pdfY: number
  pdfWidth: number
  pdfHeight: number

```

Note: `dateValue` from the CONTEXT.md decision is stored as `textValue` on the unified model (a date field's textValue is the date string). This avoids a parallel field when a single string covers both cases.

## Accessibility Requirements

All Phase 1 and Phase 2 requirements carry forward. Phase 3 additions:

- **FieldPalette buttons:** `aria-pressed="true"` on the armed button, `aria-pressed="false"` on all others. All buttons have `aria-label` (see Copywriting Contract) and 44px minimum touch target.
- **UndoRedoControls:** `aria-disabled="true"` (not HTML `disabled`) when no undo/redo history. Separate `aria-label` for disabled state (see Copywriting Contract). Same dual-label pattern (aria-label + sr-only) as Phase 1/2.
- **ZoomControl:** Zoom percentage `aria-live="polite"` so screen readers announce level changes. Zoom-out and zoom-in use `aria-disabled="true"` at limits with descriptive labels.
- **Keyboard shortcut guards:** Ctrl+Z / Ctrl+Y / Ctrl+Shift+Z undo/redo handlers MUST check `document.activeElement.tagName !== 'INPUT' && document.activeElement.tagName !== 'TEXTAREA'` before firing. Delete/Backspace field-deletion handler has the same guard (carried forward from Phase 2's equivalent requirement for any text inputs; Phase 3 makes it mandatory since text/date inputs now exist).
- **PlacedField inline inputs:** Each `<input>` has `aria-label` identifying its type and purpose (see Copywriting Contract). Inputs are in the tab order — they are reachable without requiring mouse interaction. When a user tabs into a date or text field widget, focus lands on the input.
- **CheckboxField:** Not keyboard-operable for toggling (placing = marked; no toggle). `role="img"` and `aria-label` on the widget outer wrapper identify it as a placed mark.
- **InitialsDrawModal:** Identical accessibility contract to Phase 2 SignatureDrawModal — `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to title, focus trap, Escape closes.
- **WordDocBanner:** `role="status"` and `aria-live="polite"` — informational, not urgent. "Choose a PDF instead" link has a meaningful label (not "click here"). Color contrast: `--color-text-primary` (#111827) on white — passes WCAG AA.
- **Focus rings:** All new interactive elements use the same 2px `--color-accent` outline, 2px offset pattern.
- **Color contrast (new pairs):**
  - White label on `--color-accent` (#2563EB): 4.54:1 — passes AA (armed palette button — same as Phase 1/2 primary buttons)
  - `--color-text-primary` (#111827) on white in text/date inputs: 16.75:1 — passes AA
  - `--color-text-secondary` (#6B7280) in zoom readout on white: 4.63:1 — passes AA

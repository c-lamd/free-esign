<objective>
Detect Word documents (.doc/.docx) at file selection and show a friendly "export to PDF first" guidance banner instead of attempting any conversion — closing DOC-05.

This is a fully independent vertical slice: a user who picks a .doc or .docx file immediately sees a clear, non-alarming message explaining why Word files aren't supported (formatting integrity + privacy) and how to proceed, with a one-click path back to choosing a PDF. No file shares with the field-type or zoom work, so this runs in Wave 1 in parallel with Plan 01.

Per CONTEXT.md Area 4 (detection by extension AND MIME, friendly guidance, no silent conversion) and the UI-SPEC WordDocBanner contract.

Purpose: Word files are the most common "why won't this open?" failure; a helpful prompt prevents confusion and reinforces the privacy promise.
Output: `fileValidation.ts` with a `'word-doc'` result, a new `WordDocBanner.tsx`, UploadZone routing to it, and extended `fileValidation.test.ts`.
</objective>

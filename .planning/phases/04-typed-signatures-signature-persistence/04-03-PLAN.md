## Trust Boundaries

|----------|-------------|
| user input → DOM | typed name/initials string rendered in preview, widget, saved-item card |
| user input → IndexedDB | typed text / drawn PNG persisted via addSavedItem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Denial of service | name/initials input length | mitigate | `<input maxLength={100}>` caps preview/export text (V5 input validation); drawSignatureText scales to fit, no loop |
| T-04-09 | Tampering | typed text in live preview / SavedItemCard / widget | mitigate | React renders all user text as text nodes (JSX interpolation), never dangerouslySetInnerHTML — XSS-safe on screen |
| T-04-10 | Tampering | font family selection | mitigate | font picker is a fixed 3-option radiogroup; selectedFont is one of the allowlisted names that FONT_FILE_MAP accepts (Plan 02) — no free-form font path |
| T-04-11 | Information disclosure | addSavedItem persistence failure | accept | failure is non-blocking and surfaced with the "Couldn't save this for reuse" inline copy; no data leaves the origin |
</threat_model>

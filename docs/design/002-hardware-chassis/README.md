# Sketch 002: Hardware Chassis

Follow-on to 001. Accent (**Signal Orange**) and wordmark (**free·esign**) are locked — this sketch
only explores **how much hardware** the editor should wear. Three escalating levels, switch with the
`L1 / L2 / L3` tabs in the dark sketch bar.

## Design Question
"More hardware" — but how much before it stops being a tool and starts being a toy? This is an
intensity ladder so you can stop at the right rung.

## How to View
```
open .planning/sketches/002-hardware-chassis/index.html
```

## Levels
- **L1 — Minimal** — the level from sketch 001 (flat control bar + status strip). The reference point.
- **L2 — Console** ⭐ *(my pick for the real editor)* — the editor wrapped in an aluminum **chassis**
  with corner **screws**, a glowing **LCD readout** (PG/ZM/FLD + `ORIG.BYTES OK`), a rotary **ZOOM
  knob** with a tick ring, a fuller **keypad** (numbered keys + silkscreen sub-labels), a utility row
  (PG/FIT/1:1/ROT/GRID/UNDO/REDO), **PWR/READY/SAVED LEDs**, and a speaker **grille**. Lots of device
  character, document still has full room. Practical.
- **L3 — Full Unit** — the whole app as a rendered **handheld**: thick rounded chassis, hand-strap,
  rubber feet, model badge (`ES·1 · rev.A`), the document shown on the unit's inset screen. Gorgeous
  as a *product shot / landing hero* — but the chassis eats space you'd want for actually signing.

## What to Look For
- Where does it tip from "considered instrument" → "too much"? (My read: **L2 for the working editor,
  L3 reserved for the landing hero / marketing.**)
- The **LCD** turning your `ORIGINAL BYTES PRESERVED` guarantee into a glowing readout — keep it?
- Knob vs the L1 fader for zoom — does the rotary earn its space?
- Silkscreen numbered keys (`01 SIG / sign`) vs the plain `SIG` keys — too busy, or just right?

## Recommendation (to confirm or overrule)
Ship **L2 Console** as the editor, borrow **L3** styling for the landing "unit" hero. Best of both:
the app feels like an instrument without sacrificing the working area.
